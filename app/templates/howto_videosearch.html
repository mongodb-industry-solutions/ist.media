Video Search – Behind the Scenes

We are using multimodal embeddings generated by Voyage AI.
Source file snippets below are shortened for better readability. Check the github for complete code.

Workflow for Video Ingestion

To include another video.mp4 in the search, the following steps have to be conducted.

  • Generate the mp3 voice track (video2audio)
  • Generate the transcript from the mp3 (audio2text). Create snippets of between 10-20 seconds in length.
  • Create frames (image snapshots) from the video. Create one file every 2 seconds.
  • Generate the multimodal vector embeddings with Voyage AI. Align multiple image frames to the same voice track, e.g. 5 frames (representing 10 seconds of the video) are each aligned with the same 10-second transcript snippet.

Workflow for Video Search

  • Generate text embedding for the search term. Leave the image part empty.
  • Do a nearest-neighbor search, find the best matching clip.
  • Find the beginning of the scene: move backwards in time, compare the clips and stop once the delta is larger than a threshold (e.g. the scene picture is fundamentally different, not just e.g. a slight move of the camera).
